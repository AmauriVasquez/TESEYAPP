# Ingresos V3 — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans. This plan is designed to be executed by a SINGLE agent in order (F1→F2→F3). Steps use checkbox (`- [ ]`) syntax.

**Goal:** Mostrar Cliente/Cotización en el modal de pago + decidir fiscalidad ahí (toggle requiere_cfdi y botón blindado "Agregar IVA al precio"); cobro cliente-first multi-select; KPIs de Subtotal/IVA del periodo; y la vista de impresión real de la cotización desde la tabla.

**Architecture:** F1 = presentación + toggle seguro. F2 = nuevo flujo de cobro (cliente → proyectos con saldo → 1 o varios). F3 = única operación destructiva (subir precio ×1.16), blindada (solo sin-IVA, bloqueo si facturado, confirmación, permiso, auditoría).

**Tech Stack:** Vite + React 19, Supabase/Postgres, Tailwind, shadcn/radix. **Sin tests** → verificación por tarea: `npx eslint <archivos>` (0 errores nuevos; los `catch (e)` pre-existentes en Finanzas.jsx no cuentan) + `npm run build` (éxito) + checks manuales + SQL.

**Convenciones:** rama `feat/ingresos-v3`. Migración aditiva (la aplica el controlador vía MCP o el usuario; se registra en `supabase/migrations/`). Deploy manual al final. Móvil = tarjetas. Permisos front: `usePermissions().can(modulo, accion)`. Spec: `docs/superpowers/specs/2026-06-24-ingresos-v3-design.md`.

**Datos clave:** `desglosePago(monto, aplicaIva)` en `src/lib/facturacionDisplay.js`; `empresaLabel`/`marcaLabel`/`estatusFactura`; `getCuentaLabel`/`getCuenta`/`brandingToEntidad`/`validarCobro` en `src/config/cuentasPago.js`; vista `v_proyecto_pago_progreso(proyecto_id, costo_total, total_pagado, pct_pagado)`; `SeleccionarFormatoCotizacionDialog` (props `open, onOpenChange, cotizacionId, cotizacion, modoProyecto, onEditar, onCrearNuevaVersion`); `RegistrarPagoDialog` (props `open, onOpenChange, proyectoId, proyecto, pago, onSave`); `PagoMultiProyectoDialog` (props `open, onOpenChange, onSaved`).
